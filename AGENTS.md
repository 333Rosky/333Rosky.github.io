# Repository Instructions

- Be skeptical and verify changes with local checks before finalizing.
- For medium or large repository changes, create a git commit and push it.
- Use the existing commit-message style:
  - `[UPD] Clear Commit Name` for updates.
  - `[FIX] Clear Commit Name` for fixes.
  - `[ADD] Clear Commit Name` for additions.
- Keep commit names concise, readable, and capitalized.
